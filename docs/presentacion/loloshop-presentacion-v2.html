01 El reto

De llevar el stock "de memoria" a un sistema que vende solo.

Mercancía nueva sin control, ventas físicas que no llegan a Shopify, y DMs que se enfrían. La solución no es más esfuerzo manual: es software que cierra el ciclo.

Antes

Sin control, sin escala

  • Stock "de memoria" entre dos tiendas.
  • Ventas físicas que no se descuentan de Shopify.
  • DMs que piden precio/talla y nadie alcanza a responder.
  • Cero registro de quién vendió qué y cuándo.
Con LoloShop

Un sistema que trabaja por ti

  • PWA instalable: stock por tienda, alertas, movimientos.
  • Bot de WhatsApp que consulta stock/precio en vivo.
  • Arma pedidos y cobra con Mercado Pago (tarjeta/OXXO/SPEI).
  • Anuncios Click-to-WhatsApp que llenan el chat de clientes.
02 Cómo está armado

Una sola capa de datos. Tres caras hacia afuera.

App para el equipo, bot para los clientes, anuncios para atraerlos — todos leen del mismo lugar.

Equipo

PWA de inventario

Next.js instalable. Roles, stock por tienda, alertas, entradas/salidas.

Clientes

Bot WhatsApp · N8N + Claude

Atiende, cotiza, arma pedido y manda link de pago. Claude Haiku.

Adquisición

Facebook / Instagram Ads

Click-to-WhatsApp + Conversions API hacia el bot.

05 Mejoras y optimización

Decisiones de ingeniería que bajan costo y suben confiabilidad.

No es solo que funcione: está construido para ser barato de operar, fácil de probar y difícil de romper.

🧱

Capa de datos desacoplada

Todo pasa por InventoryDataSource. Cambiar de fixtures a Shopify no toca UI ni bot.

Mantenibilidad
🧪

TDD con fixtures primero

El sistema se prueba completo sin token de Shopify ni cuentas externas. Menos sorpresas en producción.

Calidad
🧠

Claude Haiku + caching

Modelo económico para chat ($1/$5 por millón) con prompt caching y límite de turnos. ~$0.02–0.05 por conversación.

−Costo IA
🆓

Ventana de 24h de WhatsApp

Responder a clientes dentro de las 24h es gratis. El bot es reactivo ⇒ casi todo el tráfico no cuesta.

−Costo mensajería
🔐

Webhook de pago firmado

Se valida la firma del webhook de Mercado Pago: nada se marca "pagado" sin confirmación real.

Seguridad
📲

PWA instalable

Se instala como app en el celular, sin App Store. Manifest + service worker; cámara para escaneo.

Adopción
🔑

Endpoints del bot con API key

/api/bot/* protegidos por x-bot-key y excluidos del login; secretos fuera del repo.

Seguridad
📈

Conversions API

Cada venta se reporta a Meta para optimizar por compradores, no por clics — mejor retorno del gasto en ads.

+ROI ads
🩺

Monitoreo del bot

Health-check del workflow de N8N con alerta al staff si deja de responder.

Confiabilidad
06 Premortem · honestidad por adelantado

"Es 2027 y el proyecto fracasó. ¿Por qué?"

Trabajamos los modos de fallo antes de invertir, para diseñar las mitigaciones desde el día uno. Estos son los 6 riesgos a accionar ya.

01
Scope creep + sin acuerdo escrito

El riesgo #1 no es técnico: mezclar amistad y negocio sin alcance definido. Mitigación: acuerdo simple por fase y congelar alcance.

02
Prerrequisitos atorados

Sin token Shopify / cuentas, todo se queda en demo. Mitigación: checklist con responsable y fecha; iniciar verificaciones lentas ya.

03
F1 sin cerrar

Abrir bot y ads con los cimientos a medias. Mitigación: terminar Productos + Dashboard antes de F3/F4.

04
Se rompe la regla de datos

Llamar a Shopify directo bajo presión ⇒ doble fuente de verdad. Mitigación: revisión/lint que lo prohíba fuera de su capa.

05
El bot promete agotados

Stock desincronizado hasta F2. Mitigación: disponibilidad "a confirmar" + humano en el loop.

06
Confirmaciones de pago falsas

Webhook MP sin validar. Mitigación: validar firma y conciliar contra el estado real de MP.

Mapa de riesgos por categoría

Adopción

A·A El staff no usa la PWA → datos viejos → el bot miente.
M·A El bot frustra al cliente en su canal principal.
M·M Desconfianza al pagar por WhatsApp.

Técnico

M·A ShopifyDataSource no calza con la interfaz (rate limits).
M·A N8N se cae sin que nadie note.
M·M Costos de Claude por prompts largos/loops.

Negocio / Ads

A·A Scope creep hunde los tiempos de entrega.
M·M Gasto en ads sin retorno (clics, no compras).
M·M Atribución rota: no se demuestra ROI.

Leyenda P·I = Probabilidad · Impacto (A alto, M medio). Reporte completo: docs/superpowers/premortem-2026-06-28-loloshop.md.

Cómo ganas tú

Cuatro modelos. Uno recomendado.

Recomendado

Setup + mensualidad gestionada

Setup $4,000–10,000 MXN · luego $1,200–2,500 MXN/mes

Cuota única de implementación + mensualidad que cubre hosting, monitoreo, soporte y tu margen. Ingreso recurrente y predecible. Como es portafolio, puedes bajar el setup a cambio del caso de éxito.

% por venta del bot

3–5% de las ventas online generadas

Alinea tu pago con el resultado. Ideal como complemento cuando ya hay volumen; requiere medir ventas atribuidas (la Conversions API ayuda).

Renta SaaS a la medida

$1,800–3,500 MXN/mes, sin setup grande

Todo en una mensualidad más alta. Menor barrera de entrada; tú conservas el sistema.

Proyecto llave en mano

Pago único $15,000–30,000 MXN

Se construye, se entrega y el cliente opera. Sin recurrente, pero cierre rápido.
